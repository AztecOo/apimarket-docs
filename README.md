# apimarket-docs
存放api市场的相关文档

服务： services下根据不同的API服务创建目录

目录： SUMMARY.md是目录文件

书写规范： 一、SUMMARY.md

* [一级目录] （对应文章路径）
  * [二级目录] (对应文章路径)
      
        * [三级目录] (对应文章路径)
    [注]：每级目录前*号以1个tab键分割

二、文章

标题： 默认与SUMMARY.md的标题名称一致，文章中不用再次填写

时间： 默认获取创建文档的时间

内容： 以h2开始，默认右侧导航抓取h2标题

跳转： 在地址前追加文件路径/static/docs-content/

   eg:  JobResult: (/static/docs-content/services/地图/地点/地图编码.md#简介)
